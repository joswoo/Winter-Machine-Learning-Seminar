Vanishing/Exploding Gradients Problem¶

10장에서 backpropagation 알고리즘은 output layer에서 input layer로 이동하면서 error gradient를 전파

이 알고리즘은 네트워크의 각 파라미터와 관련해 cost function의 gradient를 계산해, 각 파라미터를 Gradient Descent 단계로 갱신

•하지만, 이 알고리즘이 하위 layer로 진행하면서, gradient는 점점 작아짐

-> layer를 지날 때마다 최초 값보다 작아져 값을 전달해도 의미가 없다!! -> 결과적으로 Grdient Descent 업데이트는 output layer의 연결 가중치(weights)를 거의 반영하지 않으며, 좋은 솔루션으로 수렴 X

  => Vanishing gradients problem 이라고 한다.

•그 반대로, 기울기가 커질 수 있기 때문에 많은 layer가 크게 갱신됨

-> 알고리즘은 발산(diverge)하게 됨 => Exploding gradients problem 이라고 한다. (RNN에서 주로 발생)

2010년 쯤, Xavier Glorot&Yoshua Bengio 의 "Understanding the difficulty of Traning Deep FeedForward Neural Networks"라는 paper에서 몇가지 suspects를 발견했다.

1.Logistic sigmoid 활성화 함수 2.평균 0과 표준편차 1의 정규분포를 사용하는 무작위로 초기화로 사용된 가중치 초기화(weight initialization)

즉, sigmoid 활성화 함수와 initialization scheme을 통해, 각 layer의 ouput의 분산이 input의 분산보다 훨씬 더 크다는 것을 보여줌

네트워크에서, 앞으로 나아가면, 활성화 함수가 차상위 layer에서 포화 될 때까지 각 layer이후에 분산이 계속 증가 하게 됨 -> 사실 logistic 함수가 0이 아닌 0.5의 평균을 가지는 사실로 인해 악화 됨!

Xavier and He Initialization¶

앞서 말한 논문에서, Glorot와 Bengio는 이 문제를 완화시키는 방법을 제안

예측을 할 때 앞 방향, gradient를 역 전파 할 때 반대 방향으로 적절히 흐르는 것이 필요로 함(즉 양방향으로 적절히 흐르는 것)

저자들은 신호가 적절히 흐르도록 input의 분산과 같도록 각 layer의 output의 분산이 필요하다고 주장

또한, 역 방향 시 layer를 통과하기 전후에 분산이 같아야 한다고 주장

layer가 동일한 input의 수와, output connection을 가지고 있지 않으면 확신 할 수 없지만,

실제로 잘 작동하는 것으로 입증 된 좋은 절충안을 제안

: (식) 연결 가중치는 무작위로 초기화, 여기서 n_input과 n_outputs은 가중치가 초기화 되는 input & ouput의 connections 수 (fna-in과 fan-out 이라고도 함)

-> 이 초기화 전략은 Xavier initialization & Glorot initialization 라고 함

input과 output의 연결 수가 대략 같으면 보다 간단한 방정식을 얻을 수 있다.

10장에서 이 단순화된 전략을 사용했다!!

Xavier 초기화 전략을 사용하면서, 학습 속도가 빨라 질 수 있고, 이는 딥러닝의 현재 성공을 이끌어 낸 트릭 중 하나이다.

책에 있는 링크의 논문에서는, table 11-1의 표에 나와있는 것 처럼 다양한 활성화 함수에 대해 유사한 전략을 제공

ReLU 활성화 함수에 대한 초기화 전략은 때때로 He initialization 이라고도 부른다.

10장에서 소개한, tf.layers.dense() 함수는 균일 분포(uniform distribution)인 Xavier initialization을 사용

Nonsaturating Activation Functions¶

위 논문에서, vanishing/exploding gradients 문제가 부분적으로 활성화 함수를 잘못 선택 되었기 때문에 발생했다고 했다.

-> 사람들은 생물학적 뉴런에서 대략 sigmoid 활성화 함수를 사용하면 훌륭한 선택일 것이라고 생각함

하지만, Deep Neural Network(DNN)에서 다른 활성화 함수, 특히 ReLU활성화 함수는 특히 더 잘 작동 한다는 사실이 들어남! (대게 ReLU활성화 함수는 양수 값에 대해 포화되지 않기 때문! 또한 계산 속도가 매우 빠름)

하지만 ReLU함수는 dying ReLUs 로 알려진 문제로 완벽한 함수는 아니다.

dying ReLUs란 학습 동안, 몇몇 뉴런은 실제적으로 죽게되는데, 이는 0 이외의 다른 값의 출력을 하지 않는 것을 의미

(ReLU함수는 0보다 낮은 값에서의 gradient값은 0으로만 출력 됨)

경우에 따라서는(특히 learning rate이 클수록) 네트워크 학습 뉴런의 절반이 죽어 있을 수 도 있다.

학습 도중 뉴런의 입력 가중치의 합(wight sum of the neuron's input)이 음수가 되도록, 뉴런의 가중치(neuron's weights)가 업데이트 된다면, 이는 0을 출력

이 때, 뉴런은 다시 생기지 않을 것 -> 입력이 음수 일 때, ReLU의 gradient가 0이기 때문

이러한 문제를 해결하기 위해, leaky ReLU 와 같은 변형된 ReLU함수를 사용할 수 있다.

링크 된 논문에서는 ReLU 활성화 함수의 여러 변종에 대해 비교를 했고, 결론 중 하나는 leaky ReLU()가 strict ReLU보다 우월하다는 것이다.

  1. 또한 randomized leaky ReLU(RReLU) 에 대해서도 평가했다.

여기서 α는 학습 중 주어진 범위에서 무작위로 추출되며, 테스터를 할 때에는 평균 값으로 고정

RReLU는 잘 수행 되었고, regularizer로써의 역할을 수행하는 것 처럼 보였다.

  1. 마지막으로, parametric leaky ReLU(PReLU) 를 평가

PReLU에서 α는 역전파(back propagation)에 의해 수정 될 수 있는 파라미터 값으로, 학습하는 동안 배울 수 있다.

PReLU는 큰 이미지 데이터 세트에서 ReLU보다 뛰어난 것으로 보고되었지만, 작은 이미지 데이터 셋에서는 overfitting 위험이 있다.

    return np.where(z < 0, alpha * (np.exp(z) - 1), z)

ReLU 함수와 기능이 비슷

1.z<0 일 때, 음의 값을 취함 -> vanishing gradient 문제를 완화

α는 z가 큰 음수 일 때 ELU 함수가 접근하는 값을 정의, 일반적으로 1로 설정(조정 가능) 2.z<0에 대해 0이 아닌 gradient를 가지기 때문에, dying units 문제를 피할 수 있음 3.z=0 주변을 포함하여 모든 면에서 smooth -> gradient descent의 속도를 높이는데 도움

하지만 단점도 존재한다.

1.지수 함수의 사용으로 ReLU와 그 변형에 비해 느린 계산 2.학습동안에는 더 빠른 convergence rate 로 보상 받지만, 테스트 시간에 ReLU보다 느리다.

Batch Normalization¶

Segey loffe&Christian Szegedy의 논문에서 vanishing/exploding 문제를 처리하기 위해 Batch Normalization(BN) 이라는 기술을 제안

더 일반적으로 학습 동안, 이전 layer의 파라미터가 변경됨으로써, 각 layer들의 입력 분포가 변경되는 문제를 해결

이 기법은 각 layer의 activation function 직전에 모델에 operation(연산)을 추가하고, 간단히 input값들을 zero-centering과 normalizing 한다.

그 다음 layer 당 두개의 새로운 파라미터를 사용해 결과를 scaling 하고, shifting하는 것으로 구성

즉 하나의 파라미터는 scaling용, 다른 하나는는 shifting용

다시말해, 이 operation을 통해 모델은 각 layer의 input에 대해 최적의 크기(optimal scale)와 평균을 학습할 수 있다.

input들을 중심에 맞추고 normalization하기 위해, 알고리즘은 input들의 평균과 표준편차를 추정해야한다. -> 현재 mini-batch를 통해 평균 및 표준편차를 평가

Implementing Batch Normalization with TensorFlow¶

텐서플로우는 input을 중앙에 놓고, normalize하는 tf.nn.batch_normalization() 함수를 제공하지만, 사용자가 직접 평균 및 표준편차를 계산해야 함

그리고 파라미터를 함수에 전달하고 scaling 및 offset파라미터의 생성을 처리해야 함

-> 편리하지 않다! 따라서 이 모든 것을 처리하는 tf.layers.batch_normalization()함수를 사용하면 됨!!!

Gradient Clipping¶

Gradient Clipping은 exploding gradients문제를 줄이기 위한 기법으로, 이 기법은 역전파(backpropagation)동안 임계값을 초과하지 않도록 하기 위해 gradient를 clip하는 방법

텐서플로우에서 optimizers의 minimize() 함수는 gradients 계산과 적용을 처리하므로

1.optimizer's의 compute_gradients()메소드를 먼저 호출 한 다음 2.clip_by_value() 함수를 사용하여 gradient를 clip하는 operation(연산)을 만들고 3.마지막으로 optimizer's의 apply_gradients() 메소드를 사용하여, clip된 gradient를 적용하는 operation을 만들면 된다.

MNIST를 위한 간단한 신경망을 만들고, gradient clipping을 추가해 본다.

Reusing Pretrained Layers¶

매우 큰 DNN을 처음부터 교육하는 것은 일반적으로 좋지 않다. 대신, 태클하려고하는 것과 유사한 작업을 수행하는 기존의 신경망을 찾은 다음 이 네트워크의 하위 계층을 재사용하기만 하면 되는데 이것을 이동 학습이라고합니다. 학습 속도를 크게 높일뿐만 아니라 트레이닝 데이터도 훨씬 적게 듭니다.

예를 들어 사진을 동물, 식물, 차량 및 일상 물체 등 100 가지 범주로 분류하도록 훈련 된 DNN에 액세스 할 수 있다고 가정합니다. 이제 특정 차량 유형을 분류하기 위해 DNN을 교육해야 합니다. 이러한 작업은 매우 유사하므로 첫 번째 네트워크의 일부를 재사용해야 합니다(그림 11-4)

def strip_consts(graph_def, max_const_size=32):
    """Strip large constant values from graph_def."""
    strip_def = tf.GraphDef()
    for n0 in graph_def.node:
        n = strip_def.node.add() 
        n.MergeFrom(n0)
        if n.op == 'Const':
            tensor = n.attr['value'].tensor
            size = len(tensor.tensor_content)
            if size > max_const_size:
                tensor.tensor_content = b"<stripped %d bytes>"%size
    return strip_def

def show_graph(graph_def, max_const_size=32):

    if hasattr(graph_def, 'as_graph_def'):

예를 들어 다음 코드는 숨겨진 레이어 1, 2, 3 만 복원합니다.

  • 먼저 원래 모델의 히든 레이어 1~3을 복사하여 새 모델을 만들기 위해 정규표현식 "hidden[123]"을 사용하여 히든 레이어 1~3의 모든 변수 목록을 가져옵니다.
  • 원래 모델의 각 변수 이름을 새 모델의 이름으로 매핑하는 사전을 만듭니다.
  • 이러한 변수 만 복원하는 Saver를 만듭니다. 또한 모든 변수를 초기화하는 작업을 만들고 두 번째 saver는 레이어 1에서 3까지 뿐만아니라 새 모델 전체를 저장합니다.
  • 세션을 시작하고 모델의 모든 변수를 초기화 한 다음 변수를 복원합니다 원래 모델의 레이어 1에서 3까지의 값을 반환합니다.
  • 마지막으로 새 작업에서 모델을 학습하고 저장합니다.

작업이 더 비슷할수록 더 많은 레이어를 재사용 할 수 있습니다 (하위 레이어부터 시작). 매우 유사한 작업의 경우 숨겨진 모든 레이어를 유지하고 출력 레이어를 바꿀 수 있습니다.

다른 프레임 워크를 사용하여 모델을 학습한 경우 모델 매개 변수를 수동으로 로드해야 합니다. 그런 다음 해당 매개 변수를 적절한 변수에 할당해야 하는데 매우 지루할 수 있다고 합니다. 예를 들어, 다음 코드는 다른 프레임 워크를 사용하여 훈련 된 모델의 첫 번째 숨겨진 레이어에서 weight와 bias를 복사하는 방법을 보여줍니다.

Tweaking, Dropping, or Replacing the Upper Layers¶

원본 모델의 출력 레이어는 일반적으로 새 작업에 유용하지 않을 가능성이 높기 때문에 대개 교체해야하며 새로운 작업에 대해 올바른 출력 개수가 없을 수도 있습니다.

유사하게, 원래 모델의 상위 히든 레이어는 하위 레이어만큼 유용하지는 않을 수 있다. 그 이유는 새 작업에 가장 유용한 high-level features가 원래 작업에서 가장 유용했던 features와 크게 다를 수 있기 때문입니다. 따라서 재사용 할 레이어의 수를 찾아야 합니다.

복사된 모든 레이어를 먼저 고정한 다음 모델을 교육하고 성능을 확인합니다. 그런 다음 최상위 숨겨진 레이어 중 하나 또는 두 개를 해제하여 backpropagation(역전파)을 조정하여 성능이 향상되는지 확인하십시오. 트레이닝 데이터가 많을수록 고정 해제할 수 있는 레이어가 늘어납니다.

그래도 성능이 좋지 않고 교육 데이터가 거의 없으면 상위 히든 레이어를 삭제하고 나머지 히든 레이어를 모두 다시 고정합니다. 재사용 할 레이어를 찾을 때까지 반복 할 수 있습니다. 교육 자료가 충분하다면 상위 숨겨진 레이어를 삭제하는 대신 교체하고 숨겨진 레이어를 더 추가 할 수 있습니다.

Model Zoos¶

어디서 태클하고 싶은 것과 유사한 작업을 위해 훈련된 신경 네트워크를 찾을 수 있을까? 첫 번째로 살펴볼 것은 분명히 자신의 모델 카탈로그에 있습니다. 이것은 모든 모델을 저장하고 구성하여 나중에 쉽게 검색 할 수 있는 좋은 이유 중 하나입니다. 또 다른 옵션은 Model Zoos에서 검색하는 것입니다. 많은 사람들이 다양한 작업을 위해 머신러닝 모델을 트레이닝하고 사전 트레이닝된 모델을 일반 대중에게 공개합니다.

TensorFlow는 https://github.com/tensorflow/models 에서 자체 Model Zoo을 제공합니다. 특히 VGG, Inception, ResNet (제 13 장 참조, 모델/슬림 디렉토리 확인)과 같은 최첨단 이미지 분류넷을 포함하며 코드, 사전 모델 및 인기있는 이미지 데이터 세트를 다운로드 할 수 있습니다.

또 다른것은 Caffe의 Model Zoo입니다. 다양한 데이터 세트(ImageNet, 장소 데이터베이스, CIFAR10 등)에 대한 교육을받은 많은 컴퓨터 비전 모델 (LeNet, AlexNet, ZFNet, GoogLeNet, VGGNet)을 포함합니다. https://github.com/ethereon/caffe-tensorflow 에서 다운로드 할 수 있습니다.

Unsupervised Pretraining¶

복잡한 작업을 수행하고 싶은데 비슷한 작업으로 훈련된 모델을 찾을 수 없다고 가정해 보겠습니다. 물론 더 많은 교육용 데이터를 수집해야 하지만 너무 어렵거나 너무 비싸면 Unsupervised Pretraining을 수행 할 수 있습니다(그림 11-5 참조). 즉, 레이블이 없는 트레이닝 데이터가 충분하다면 RBM(Restricted Boltmann Machines) 또는 autoencoders(15 장 참조)와 같은 unsupervised feature detector 알고리즘을 사용합니다.

각 레이어는 이전에 훈련된 레이어의 출력에 대해 트레이닝 됩니다(훈련된 레이어를 제외한 모든 레이어는 고정되어 있음). 모든 레이어를 이와 같은 방법으로 트레이닝하면 감독학습(backpropagation)을 사용하여 네트워크를 fine-tune할 수 있습니다.

이는 다소 길고 지루한 과정이지만 종종 잘 작동합니다. 실제로 Geoffrey Hinton과 그의 팀이 2006 년에 사용한이 기술은 신경망의 부활과 깊은 학습의 성공으로 이어졌습니다. 2010 년까지는 감독되지 않은 사전 훈련(일반적으로 RBM 사용)이 신경망의 표준이었으며, vanishing gradients 문제가 완화된 후에 역전파를 사용하여 DNN을 훈련하는 것이 훨씬 더 보편화 되었습니다.

그러나 Unsupervised Pretraining(RBM이 아닌 autoencoders 사용)은 해결해야 할 복잡한 작업이 있고 재사용 할 수 있는 유사한 모델이 없으며 레이블이 없는 트레이닝 데이터가 많습니다.

Pretraining on an Auxiliary Task¶

마지막 옵션은 레이블된 트레이닝 데이터를 쉽게 얻거나 생성할 수있는 보조 작업에서 첫 번째 신경망을 학습한 다음 실제 작업을 위해 해당 네트워크의 하위 레이어를 재사용하는 것입니다. 첫 번째 신경망의 하위 레이어는 두 번째 신경망에서 재사용 할 수 있는 feature detectors를 배웁니다.

예를 들어, 얼굴을 인식 할 수 있는 시스템을 구축하려는 경우 각 개인에 대해 몇 장의 이미지만 표시 할 수 있습니다. 각 사람의 사진 수백 장을 모으는 것은 실용적이지 못합니다. 그러나 인터넷에서 임의의 사람들의 사진을 많이 수집하고 두 개의 다른 사진이 동일한 사람의 feature를 찾는지 여부를 감지하는 첫 번째 신경망을 훈련할 수 있습니다. 이러한 네트워크는 얼굴에 대한 우수한 기능 감지기를 학습하므로 하위 레이어를 다시 사용하면 트레이닝 데이터를 거의 사용하지 않고도 훌륭한 얼굴 분류기를 학습할 수 있습니다.

레이블이 없는 트레이닝 사례를 수집하는 것은 종종 저렴하지만 레이블을 지정하는데 비용이 많이 듭니다. 이 경우 일반적인 기술은 모든 트레이닝 예제를 "우수"로 표시 한 다음 좋은 예제를 손상시켜 많은 새로운 트레이닝 인스턴스를 생성하고 이러한 손상된 인스턴스를 "나쁜"레이블로 지정하는 것입니다. 그런 다음 첫 번째 신경망을 인스턴스가 좋은지 나쁜지 분류하게 트레이닝 시킬 수 있습니다.

예를 들어, 수백만 개의 문장을 다운로드하여 "좋다"라고 표시한 다음 각 문장에서 임의로 단어를 변경하고 결과 문장에 "나쁜"이라고 표시할 수 있습니다. 신경망에서 "개가 잔다" 하지만 "개들은" 나쁜 문장이 될 것입니다. 하위 레이어를 재사용하면 많은 언어 처리 작업에서 도움이 될 것입니다.

또 다른 접근 방법은 각 트레이닝 인스턴스에 대한 점수를 출력하도록 첫 번째 네트워크를 하트레이닝고 좋은 인스턴스의 점수가 나쁜 인스턴스의 점수보다 적어도 약간의 마진만큼 큰지 확인하는 비용 함수를 사용하는 것입니다. 이를 max margin learning라고 합니다.

Momentum optimization¶

볼링 공이 매끄러운 표면의 완만한 경사면에서 굴러 떨어지는 것을 상상해보면, 공은 마찰이나 공기 저항에 의해 terminal velocity에 도달 할 때까지 빠르게 굴러간다.

이것은 1964년 Boris가 제안한 Momentum optimization의 간단한 아이디어다.

이와 반대로 regular Gradient Descent는 경사면을 따라 조금씩 규칙적인 단계를 거치므로 바닥에 도달하는 데 더 많은 시간이 걸릴 것이다.

Gradient Descent 방정식은 이전 Gradient가 무엇인지 전혀 신경쓰지 않는다. local gradient가 작으면 매우 느리게 진행된다.

Momentun optimization은 이전 gradient가 무엇인지에 대해 영향을 받는다. 각 반복에서 momentum 벡터 m에서 local gradient를 뺀 다음 이 momentum벡터를 단순히 추가하여 가중치를 업데이트한다. 따라서, gradient는 가속도가 아니라 속도를 이용한다. 어떤 종류의 마찰 mechanism을 시뮬레이션하고 운동량이 너무 커지지 않도록 하기 위해서 알고리즘은 0과 1사이에서 설정해야 하는 momentum이라는 새로운 파라미터 β를 도입한다. 일반적인 momentum 값은 0.9이다.

Gradient가 일정하게 유지되면 terminal velocity가 learning rate에 1/(1-β)를 곱한 값에 gradient를 곱한것과 동일한지에 대해 쉽게 확인 가능하다.

예를 들어,β=0.9인 경우 terminal velocity는 gradient의 10배와 learning rate를 곱한 것과 같기때문에, momentum optimization은 Gradient Descent보다 10배 빠르다.

따라서, momentum optimization은 Gradient descent보다 훨씬 빠르게 plateaus에서 벗어날 수 있다.

우리는 4장에서 입력 값이 매우 다른 경우에 cost function이 elongated bowl처럼 보이는 것을 확인했다. gradient descent는 가파른 경사면에서 아주 빨리 내려가지만, valley를 내려가려면 매우 오랜 시간이 걸린다. 반대로 momentum optimization의 경우 optimum에 도달할 때까지 빠르게 진행된다. Deep neural network의 경우 Batch Normalization을 사용하지 않기 때문에, 상위 layer들은 종종 다른 스케일의 입력을 가진다. 이때 momentum optimization이 큰 도움이 된다.

momentum optimization의 한가지 단점은 튜닝할 파라미터가 추가된다는 것이다. 그러나 momentum값 0.9는 일반적으로 잘 작동하고 거의 항상 gradient descent보다 빠르다.

Nesterov Accelerated Gradient¶

1983년 Nesterov가 제안한 momentum optimization의 변형은 vanilla momentum optimization보다 항상 빠르다.

Nesterov Momentum optimization 또는 Nesterov Accelerated Gradient(NAG)는 비용함수의 기울기를 local position이 아닌, momentum의 방향으로 약간 앞당기는 아이디어다.

기울기가 Θ가 아닌 Θ+βm에서 측정된다는 점이 vanilla momentum optimization과의 유일한 차이점이다.

이러한 미세 조정은 일반적으로 momentum벡터가 올바른 방향으로 향하기 때문에 효과적이다. 그림 11-6에서 볼수 있듯이 원래 위치에서 gradient를 사용하는 대신에 그 방향으로 조금 더 멀리 측정한 gradient를 사용하는 것이 좀 더 정확하다.

In [79]:
optimizer = tf.train.MomentumOptimizer(learning_rate=learning_rate,
                                       momentum=0.9, use_nesterov=True)

NAG는 일반적인 momentum optimization에 비해 거의 항상 학습 속도를 향상시킨다. 이를 사용하려면 MomentumOptimizer를 만들 때 use_nesterov=True로 설정한다.

AdaGrad¶

Gradient Descent는 가파른 경사면을 빠르게 진행하며 천천히 valley 아래쪽으로 내려간다. 알고리즘은 초기에 이를 감지하고 그 방향을 수정하여 global optimum값을 향해 좀 더 잘 맞도록 방향을 수정하면 좋을 것이다.

AdaGrad알고리즘은 가장 가파른 차원을 따라 gradient 벡터를 축소함으로써 이를 달성한다.

첫번째 단계는 Gradient의 제곱을 벡터 s에 축적한다. 이 vectorized form은 벡터 s의 각 요소 si에 대해 (299p 첫째줄 식)을 계산하는 것과 같다.

두번째 단계는 Gradient Descent와 거의 동일하지만 큰 차이점이 있다. Gradient벡터는 루트(s+ε)의 인수로 축소된다. 이 vectorized form은 모든 매개면수 Θi에 대해 (299p 9번째 줄 식)을 계산하는 것과 같다.

다시말해서, 이 알고리즘은 가파른 차원에서 더 빠르다.

이를 adaptive learning rate라고 한다. 그림 11-7과 같이 이는 resulting updates를 global optimum을 향해 보다 직접적으로 가리킬 수 있다. 또한 추가적인 이점으로 learning rate 파라미터 n의 튜닝이 훨씬 덜 필요하다.

AdaGrad는 종종 단순한 2차식 문제에 대해 잘 수행하지만, 불행히도 신경망을 훈련할 때 너무 일찍 종료되는 경우가 많다. learning rate가 너무 낮아져서 알고리즘이 global optimum에 도달하기 전에 완전히 멈춘다. 따라서 tensorflow에 AdagradOptimizer가 있더라고 deep neural network를 학습하는 데 사용해서는 안된다. 선형회귀와 같은 간단한 작업에서는 효율적이다.

RMSProp¶

AdaGrad가 느려지고 global optimum으로 수렴하지 못하는 경우에도 RMSProp알고리즘은 가장 최근 반복의 gradient만 축적함으로써 이를 수정한다. 이것은 첫 번째 단계에서 exponential decay를 사용하여 수행된다.

감쇠율(decay rate)는 일반적으로 0.9로 설정된다. 이것은 새로운 파라미터이다. 그러나 β값 0.9는 일반적으로 잘 동작하기 때문에 조정할 필요가 전혀 없을 수 있다.

Adam Optimization¶

Adaptive momentestimation을 나타내는 Adam은 momentum optimization과 RMSProp의 아이디어를 결합하였다.

(식 11-8 참조) 1,2,5번만 보면 Adam optimization이 Momentum optimization과 RMSProp과 유사하다는 것을 알 수 있다. 유일한 차이점은 1번에서 exponentially decaying sum이 아닌 exponetial decaying average를 계산한다는 것이다. 이는 상수를 제외하고는 동일하다. 3,4번은 기술적인 세부사항이다.

Learning Rate Scheduling¶

좋은 learning rate를 찾는 것은 다소 까다롭다. 너무 높게 설정하면 발산한다. 또한 너무 낮게 설정하면 학습은 결국 optimum으로 수렴하지만 매우 오랜 시간이 걸린다. 약간 높게 설정하면 처음에는 매우 바르게 진행하지만, optimum근처에서 여기저기 움직이며 끝나지 않는다.(그림 11-8 참고) 조금의 epoch동안, 다양한 learning rate를 사용하고 학습 곡선을 비교하는 네트워크를 여러번 학습하면 상당히 좋은 learning rate를 찾을 수 있다. 이상적인 learning rate는 빠르게 학습하고 좋은 솔루션으로 수렴할 것이다.

그러나, constant learning rate보다 더 잘할 수 있다. 만약 높은 learning rate로 시작한 다음, 빠르게 진행하지 않을 때, learning rate를 줄이면 최적의 constant learning rate보다 빠른 속도로 좋은 솔루션에 도달할 수 있다.

학습 도중 learning rate를 줄이기 위한 여러가지 전략이 있는데 이를 learning schedules라고 부른다.

1) Predetermined piecewise costant learning rate 2) Performance scheduling 3) Exponential scheduling 4) Power scheduling

tensorflow로 learning schedule을 구현하는 것은 매우 간단하다.

Avoiding Overfitting Through Regularization¶

심층 신경망은 일반적으로 수천, 수만 가지 파라키터를 가지고 있다. 많은 파라미터를 가진 네트워크는 엄청난 양의 자유도를 가지며 매우 다양한 종류의 복잡한 데이터 세트에 적합하다. 하지만 이런 큰 유연성은 학습세트를 overfitting할 수도 있다.

따라서 이번 섹션에서는 신경망에 가장 널리 사용되는 정규화 기술과 텐서플로우를 이용하여 구현하는 방법에 대해 소개한다.

Early Stopping¶

학습 셋의 overfitting을 피하기 위한 방법으로 early stopping이 있다. 이 방법은 유효성 검사set의 성능이 떨어지기 시작할 때 학습을 중단시킨다.

텐서플로우로 이를 구현하는 방법 중 하나는 정기적으로 유효성 검사 set에서 모델을 평가하고 이전의 winner스냅샷 보다 좋을 떄 winner 스냅샷을 저장하는 것이다. 마지막 winner 스냅샷이 저장 된 이후의 step의 수를 계산하고 이 수치가 일부 제한에 도달하면 학습을 중단한다. 그 후 마지막 winner 스냅샷을 복원하면 된다.

early stopping은 보통 다른 정규화 기술과 결합하여 네트워크에서 훨씬 더 높은 성능을 얻을 수 있다.

Dropout¶

심층 신경망에 가장 널리 사용되는 조직화 기법은 dropout이다. 이 방법은 매우 성공적이다. 최신식의 신경망에서도 dropout을 추가하여 1-2%의 정확도 향상을 얻었다.

이 알고리즘은 매우 간단한데 모든 학습단계에서 모든 뉴런은 일시적으로 dropped out되는 확률 p를 갖는다. 이는 학습 단계에서 완전히 무시되는 것을 의미하지만, 아래 그림과 같이 다음 단계 동안은 활성화 될 수 있다. 파라미터 p는 dropout rate라고 불리며 일반적으로 50%로 설정된다. 학습 후에는 뉴런이 더 이상 dropped되지 않는다.

dropout은 각 학습 단계에서 고유한 신경망이 생성된다. 각 뉴런은 존재하거나 부재일 수 있기 때문에 가능한 뉴런의 수는 총 2N개이다.(N은 droppable한 뉴런의 총 개수) 동일한 신경망이 두 번 샘플링 되는 것은 사실상 불가능한 아주 큰 숫자인데 10,000번의 학습 step으로 run하면 10,000개의 다른 신경망이 학습된다. 이러한 신경망은 많은 가중치를 공유하기 때문에 독립적이지는 않지만 모두 다르다. 신경망의 결과는 이러한 모든 작은 신경망의 평균 앙상블로 평균 낼 수 있다.

위의 신경망은 작지만 중요한 기술적 세부사항이 있는데 p=50% 라고 가정하면 테스트 중에 뉴런은 학습 중, 평균적으로 입력 뉴런의 두 배 보다 많이 연결된다. 이러한 사실을 보완하기 위해 학습 후 각 뉴런의 입력 연결 가중치를 0.5로 곱해야 한다. 이렇게 보완하지 않으면 각 뉴런은 네트워크가 학습 된 것보다 약 2배 큰 총 입력신호를 얻게 될 것이므로, 성능이 좋지 않을 것이다. 일반적으로 학습 후에 각 입력 연결 가중치에 keep probability(1-p)를 곱해야 한다.

대안으로 각 뉴런의 출력을 학습 중에 (1-p)로 나눌 수 있다. 텐서플로우를 사용하여 dropout을 구현하려면 입력 layer 과(/또는) 원하는 hidden layer의 출력에 tf.layers.dropout()을 적용하면 된다.

batch nomalization을 수행 한 것처럼 학습을 할 때에는 training을 True로 설정하고, test를 할 때에는 기본 값인 False값을 그대로 둔다. 모델이 overfitting 되면 dropout rate를 높이고 모델이 underfit인 경우에 dropout rate를 줄여야한다. 또한 large layers에 대해서는 dropout rate를 높이고 small한 경우에 dropout rate를 줄일 수 있다. dropout은 최적의 값에 수렴하기에 상당히 늦어질 수 있지만 일반적으로 제대로 조정하면 훨씬 좋은 모델이 될 수 있다.

Max norm¶

신경망에서 널리 사용되는 또 다른 regularization 기법으로 max-norm regularization이 있다. 이 기법은 각 뉴런에 대해 incoming connections의 가중치 w를 ||W||_2 < = r과 같이 제한된다. (r은 max-norm 파라미터, ||||_2는 ℓ1 norm 이다.) r을 줄이면 정규화의 양이 늘어나고 overfitting을 줄이는데 도움이 된다.

def max_norm_regularizer(threshold, axes=1, name="max_norm",
                         collection="max_norm"):
    def max_norm(weights):
        clipped = tf.clip_by_norm(weights, clip_norm=threshold, axes=axes)
        clip_weights = tf.assign(weights, clipped, name=name)
        tf.add_to_collection(collection, clip_weights)
        return None # there is no regularization loss term
    return max_norm
