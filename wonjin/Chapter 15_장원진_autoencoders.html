Chapter 15. Autoencoders¶

autoencoder는 coding이라고 불리는 input data의 효율적인 표현을 학습하는데 유용한 인공신경망이다. 이런 coding들은 전형적으로 input data보다 훨씬 더 적은 차원을 가진다. 그리고 이것은 autoencoder를 차원축소에 있어서 유용하게 만들어준다. 더 중요한것은, autoencoder는 강력한 feature detector로 동작하고 그들은 deep neural networks의 비지도 pretraining에 사용될 수 있다. 그들은 랜덤하게 training data와 비슷한 새로운 data를 생성할 수 있고 이것을 generative model이라고 부른다. autoencoder는 input을 output에 복사하는 법을 배움으로써 동작한다. coding은 몇몇 제약조건아래에 identity function을 학습하기 위한 autoencoder의 시도의 부산물이다.

이번 장에서 우리는 어떻게 autoencoder가 동작하고, 어떤 유형의 제약조건이 부과되고, 그들을 Tensorflow상에서 어떻게 구현하는지 알아볼 것이다.(차원축소,feature extraction, unsupervised-pretraining,generative models)

처음 위의 시퀀스들을 봤을 때 첫번째 시퀀스가 짧아서 더 외우기 쉬워보인다. 하지만, 두번째 시퀀스를 자세히 들여다보면 우리는 두가지 규칙을 발견할수 있다. 첫번째는 짝수의 경우 절반의 값이 바로 뒤에 나온다는 점, 두번째는 홀수의 경우 그 수의 3배에 1을 더한 수가 다음에 나온다는 점이다. 우리가 이러한 패턴을 발견하면 첫번째 시퀀스보다 두번째 시퀀스가 훨씬 더 외우기 쉬울것이다. 패턴을 인식하는데 도움이 되는 긴 시퀀스를 외우는 것이 어렵다라는 사실은 학습도중 autoencoder를 제한하는 것이 데이터의 패턴을 발견하고 악용하도록 하는 이유가 무엇인지 분명히 설명해준다.

기억,인지,그리고 패턴 matching사이의 관계는 Willam과 Herbert가 1970년대 초기에 연구했다. 그들은 전문 체스 선수들이 5초동안 보드를 바라봄으로써 게임에서의 모든 말의 위치를 기억할 수 있다는것을 발견했다. 그러나, 이것은 오직 말들이 현실적인 위치에 놓였을때만 가능하다. 체스 전문가들은 우리보다 더 나는 기억력을 가진것이 아니다. 그들은 단지 게임에서의 그들의 경험 덕분에 더 쉽게 체스 패턴을 보는것이다. 패턴을 알아내는것은 그들이 정보를 더 효율적으로 저장하게 한다.

기억실험에서 체스 선수들처럼, autoencoder는 input들을 바라보고, 그것들을 효율적인 내부의 표현으로 바꾸고, input과 가장 유사한 어떤 것을 내보낸다. autoencoder는 encoder와 decoder 두 부분으로 나눠져있다.

그림 15-1에서 보면 autoencoder는 10장에서 배운 Multi-Layer Perception와 output layer의 뉴런의 수가 input의 수와 같다는 점을 제외하고는 같은 아키텍쳐를 가지고있다. autoencoder가 input를 재구성하고 비용함수는 reconstruction loss를 포함하기 때문에 output은 종종 reconstructions라고 불린다.

내부 표현은 input data보다 더 적은 차원을 가지기때문에 autoencoder는 undercomplete라고 말하기도한다.

2. Perfoming PCA with an Undercomplete Linear Autoencoder¶

만약 autoencoder가 오직 linear activation만을 사용하고 비용함수가 Mean Squared Error라면, 결국 이것은 PCA를 수행하게된다.

다음 코드는 PCA를 수행하기 위해 선형의 autoencoder를 만든다.

3. Stacked Autoencoders¶

다른 neural network들처럼 autoencoder는 다양한 hidden layer들을 가지고 있다. 이 경우에 그들은 stacked autoencoders라고 불린다. 더 많은 layer들을 추가하는 것은 autoencoder가 더 복잡한 coding을 학습할 수있도록 한다. 그러나 autoencoder가 너무 강력해지지 않도록 해야한다.

stacked autoencoder의 아키텍쳐는 중앙의 hidden layer에 대해 대칭적이다. 이것은 샌드위치와 비슷한 형태이다.

reset_graph()

from functools import partial

n_inputs = 28 * 28
n_hidden1 = 300
n_hidden2 = 150  # codings
n_hidden3 = n_hidden1
n_outputs = n_inputs

learning_rate = 0.01
l2_reg = 0.0001

X = tf.placeholder(tf.float32, shape=[None, n_inputs])

he_init = tf.contrib.layers.variance_scaling_initializer() # He initialization
#Equivalent to:
#he_init = lambda shape, dtype=tf.float32: tf.truncated_normal(shape, 0., stddev=np.sqrt(2/shape[0]))
l2_regularizer = tf.contrib.layers.l2_regularizer(l2_reg)
my_dense_layer = partial(tf.layers.dense,
                         activation=tf.nn.elu,
                         kernel_initializer=he_init,
                         kernel_regularizer=l2_regularizer)

hidden1 = my_dense_layer(X, n_hidden1)
hidden2 = my_dense_layer(hidden1, n_hidden2)
hidden3 = my_dense_layer(hidden2, n_hidden3)
outputs = my_dense_layer(hidden3, n_outputs, activation=None)

reconstruction_loss = tf.reduce_mean(tf.square(outputs - X))

reg_losses = tf.get_collection(tf.GraphKeys.REGULARIZATION_LOSSES)
loss = tf.add_n([reconstruction_loss] + reg_losses)

optimizer = tf.train.AdamOptimizer(learning_rate)
training_op = optimizer.minimize(loss)

init = tf.global_variables_initializer()
saver = tf.train.Saver() # not shown in the book

Now let's train it! Note that we don't feed target values (y_batch is not used). This is unsupervised training.

3.2 Tying weights¶

autoencoder가 대칭적일 때, 공통 기술은 decoder layer의 가중치를 encoder layer의 가중치에 엮는것이다. 이것은 모델의 가중치의 수를 반으로 줄여주고 학습의 속도를 높이고 overfitting의 위험을 줄여준다. TensorFlow에서 dense()함수를 사용하여 연결된 가중치를 실행하는 것은 조금 다루기힘들다. layer들을 직접 정의하는 것이 더 쉽다.

3.3 Training one Autoencoder at a time¶

전체 stacked autoencoder를 한번에 학습하는 것 대신에 한번에 하나의 얕은 autoencoder만 학습시키는 것이 더 빠르다. 이것은 종종 매우 깊은 autoencoder들에 사용된다.

[그림 15-4 참고] 학습의 첫번째 단계에서, 첫번째 autoencoder는 input을 재구성하는 것을 배운다. 두번째 단계에서, 두번째 autoencoder는 첫번째 autoencoder의 hidden layer의 output을 재구성하는 것을 배운다. 우리는 최종적으로 그림 15-4에서 보이는 것처럼 모든 autoencoder를 사용하여 커다란 샌드위치를 만들 수 있다.

이런 다 단계의 학습 알고리즘을 실행하기위한 가장 단순한 접근법은 각 단계마다 다른 TensorFlow 그래프를 사용하는 것이다. autoencoder를 학습한 후에, 우리는 이것을 통해 training set을 동작시키고 hidden layer의 출력을 확인할 수 있다. 이 출력은 다음 autoencoder를 위한 training set으로 활용된다. 모든 autoencoder들이 이런 방식으로 학습되면, 우리는 가중치와 bias들을 각각의 autoencoder로부터 복사해올수 있고 그것들을 stacked autoencoder를 만드는데에 사용할 수 있다.

또 다른 접근법은 각 학습 단계를 수행하기위해 몇개의 작업을 더하여 전체 stacked autoencoder를 포함하는 단일의 그래프를 사용하는 것이다. (그림 15-5참고) 그래프의 중앙 열은 full stacked autoencoder이다. 이 부분은 트레이닝 후에 사용될 수 있다.

왼쪽 열은 학습의 첫번째 단계를 수행하는데 필요한 일련의 작업이다. 이것은 hidden layer 2와 3을 우회하는 output layer이다.

그래프의 오른쪽 열은 학습의 두번째 단계를 수행하는데 필요한 일련의 작업이다. 이것은 hidden layer 3의 output을 가능한한 hidden layer 1의 output에 가깝게 하도록 하는 학습작업을 추가한다. 우리는 단계2를 실행하는 동안 hidden layer 1을 동작시키지 않는다.

optimizer = tf.train.AdamOptimizer(learning_rate)

with tf.name_scope("phase1"):
    phase1_outputs = tf.matmul(hidden1, weights4) + biases4  # bypass hidden2 and hidden3
    phase1_reconstruction_loss = tf.reduce_mean(tf.square(phase1_outputs - X))
    phase1_reg_loss = regularizer(weights1) + regularizer(weights4)
    phase1_loss = phase1_reconstruction_loss + phase1_reg_loss
    phase1_training_op = optimizer.minimize(phase1_loss)

with tf.name_scope("phase2"):
    phase2_reconstruction_loss = tf.reduce_mean(tf.square(hidden3 - hidden1))
    phase2_reg_loss = regularizer(weights2) + regularizer(weights3)
    phase2_loss = phase2_reconstruction_loss + phase2_reg_loss
    train_vars = [weights2, biases2, weights3, biases3]
    phase2_training_op = optimizer.minimize(phase2_loss, var_list=train_vars) # freeze hidden1

"첫번째 단계" 우리는 hidden layer 2와 3을 거치지 않는 output layer를 만든다. 그리고 output과 input사이의 거리를 최소화하기위한 학습 작업을 만든다.

4. Unsupervised Pretraining Using Stacked Autoencoders¶

만약 우리가 복잡한 지도 학습을 다루지만 많은 라벨화된 training data가 없다면, 한가지 해결책은 비슷한 작업을 수행하는 neural network를 찾고 그것의 낮은 layer들을 재사용하는 것이다. 이것은 우리의 neural network가 저수준의 feature들을 모두 학습할 필요가 없기때문에 적은 training data를 사용하여 높은 수준의 모델을 학습시키는 것이 가능하다. 이것은 기존의 net에서 학습된 feature detector들을 재사용할 것이다. 유사하게, 만약 큰 dataset이지만 대부분이 라벨화되어있지않다면, 첫번재로 모든 데이터를 사용하여 stacked autoencoder를 학습하고 우리의 실제 작업을 위한 neural network를 만들기위해 가장 낮은 layer들을 재사용할 수 있다. 그리고 라벨화된 데이터를 사용하여 그것을 학습시킬 수 있다.

그림 15-8은 classification neural network를 위한 unsupervised pretraining을 수행하기 위해 어떻게 stacked autoencoder를 사용할지 보여준다.

TensorFlow 실행에 있어 특별한 것은 없다. 단지 autoencoder를 모든 training data를 사용하여 학습시키고 새로운 neural network를 만들기 위해 이것의 encoder layer들을 재사용하는 것이다.

5. Denoising Autoencoders¶

6. Sparse Autoencoders¶

좋은 feature 선택으로 이끄는 또다른 제약조건의 종류는 sparsity이다. 적절한 term을 비용함수에 추가함으로써, autoencoder는 coding layer의 활성화 뉴런의 수를 줄이기 위해 푸쉬된다. 이로 인해 autoencoder는 각 입력을 작은 수의 활성화 조합으로 나타내야한다. 그 결과, coding layer의 각 뉴런은 유용한 feature를 나타내게 된다.

분산 모델을 선택하기 위해, 우리는 처음으로 각 학습 단계에서 coding layer의 실제 희소성을 측정해야한다. 일단 우리가 뉴런당 mean activation을 가지면, 우리는 비용함수에 sparsity loss를 추가함으로써 너무 많이 활성화된 뉴런들을 사용하지 않는다. 한가지 방법은 단순히 제곱 오차를 비용함수에 더하는 것인데 실제로 더 나은 방법은 Kullback-Leibler divergence를 사용하는 것이다. 이것은 그림 15-10에서 보는 것처럼 Mean Squared Error보다 더 강한 gradient를 가진다.

def kl_divergence(p, q):
    # Kullback Leibler divergence
    return p * tf.log(p / q) + (1 - p) * tf.log((1 - p) / (1 - q))

learning_rate = 0.01
sparsity_target = 0.1
sparsity_weight = 0.2

X = tf.placeholder(tf.float32, shape=[None, n_inputs])            # not shown in the book

hidden1 = tf.layers.dense(X, n_hidden1, activation=tf.nn.sigmoid) # not shown
outputs = tf.layers.dense(hidden1, n_outputs)                     # not shown

hidden1_mean = tf.reduce_mean(hidden1, axis=0) # batch mean
sparsity_loss = tf.reduce_sum(kl_divergence(sparsity_target, hidden1_mean))
reconstruction_loss = tf.reduce_mean(tf.square(outputs - X)) # MSE
loss = reconstruction_loss + sparsity_weight * sparsity_loss

optimizer = tf.train.AdamOptimizer(learning_rate)
training_op = optimizer.minimize(loss)

7. Variational Autoencoder¶

Variational Autoencoder는 우리가 다룬 모든 autoencoder와 조금 다르다.

  • 그들은 probabilistic autoencdoers이다. 이것은 그들의 output이 부분적으로 우연히 결정되었다는 것을 의미한다.
  • 그들은 generative autoencoders이다. 이것은 그들이 학습 셋으로부터 표본화 된 것처럼 보이는 새로은 인스턴스들을 만들수 있다는 것을 의미한다.

그림 15-11의 왼쪽은 variational autoencoder를 보여준다. 기본구조는 동일하지만, encoder는 mean coding 뮤와 standard deviation 시그마를 생성한다. 실제 coding은 평균 뮤와 표준편차 시그마를 사용한 가우시안 분포로 표본화된다. 그 이후에 decoder가 표본화된 coding을 디코딩한다. 그림의 오른쪽은 이 autoencoder를 거친 학습 인스턴스를 보여준다.

input이 나선형의 분포를 가지더라도, variational autoencoder는 그들이 단순 가우시안 분포로부터 표본화된것처럼 보이는 coding을 생성한다. 학습하는 동안,비용 함수는 coding을 서서히 이동시켜 가우시안 점의 구름처럼 보이는 구형의 영역을 차지하도록 coding 공간을 밀어 낸다.

한가지 엄청난 결과는 variational autoencoder를 학습시킨후에 우리는 쉽게 새로운 인스턴스를 생성할 수 있다는 것이다.

비용함수는 크게 두가지 부분으로 나눠져있다.

첫번째는 usual reconstruction loss이다. 이것은 autoencoder가 input를 재생산하도록 한다. 두번째는 latent loss이다. 이것은 autoencoder가 단순 가우시안 분포로부터 표본화된것처럼 보이는 coding을 가지도록한다.

8. Other Autoencoders¶

autoencoder와 다른 비지도 학습알고리즘의 새로운 아키텍쳐들은 계속 개발되고있다.

  • Contractive autoencoder(CAE) 두개의 비슷한 input은 비슷한 coding을 가져야한다.

  • Stacked convolutional autoencoders convolutional layer를 통해 이미지 처리를 재구성함으로써 시각적 feature들을 뽑아내는것을 학습하는 autoencoder

  • Generative stochastic network(GSN) denoising autoencoder의 일반화

  • Winner-take-all(WTA) autoencoder 이것은 sparse coding으로 이끈다. 비슷한 WTA 접근법은 sparse convolutional autoencoder를 만드는데에 사용될 수 있다.

  • Adversarial autoencoders 한 네트워크는 이것의 input를 재생산하기위해 학습된다. 그리고 동시에 또다른 것은 첫번째 네트워크가 적절히 재구성하지 못한 input를 찾기위해 학습된다.
